## gitdb provides the following functionality

- GitHub token is provided via "--github-token" option at command execution time. If not provided, it is read from the environment variable "GITDB_GITHUB_TOKEN". If this is empty, it will be used without a token.

## Main dependencies

- octocrab: GitHub API client
- duckdb: Embedded analytical database for storing and querying repository data

### MCP & Client

#### As MCP server

Defined in /bin/gitdb_mcp.rs and under /tools directory

### As client command

The binary /bin/gitdb_cli.rs is executed.

## Database specifications (DuckDB)

### Database schema

The system uses DuckDB as an embedded database to store repository metadata, commit history, and search results.

#### Tables

1. **repositories**
   - id: INTEGER PRIMARY KEY
   - owner: VARCHAR
   - name: VARCHAR
   - full_name: VARCHAR
   - description: TEXT
   - stars: INTEGER
   - forks: INTEGER
   - language: VARCHAR
   - created_at: TIMESTAMP
   - updated_at: TIMESTAMP
   - indexed_at: TIMESTAMP

2. **commits**
   - sha: VARCHAR PRIMARY KEY
   - repository_id: INTEGER REFERENCES repositories(id)
   - author_name: VARCHAR
   - author_email: VARCHAR
   - message: TEXT
   - timestamp: TIMESTAMP
   - parent_sha: VARCHAR

3. **files**
   - id: INTEGER PRIMARY KEY
   - repository_id: INTEGER REFERENCES repositories(id)
   - path: VARCHAR
   - content: TEXT
   - size: INTEGER
   - last_modified: TIMESTAMP

4. **search_cache**
   - id: INTEGER PRIMARY KEY
   - query: VARCHAR
   - search_type: VARCHAR
   - results: JSON
   - created_at: TIMESTAMP
   - expires_at: TIMESTAMP

### Database operations

- Connection management: Automatic connection pooling with configurable limits
- Query optimization: Uses prepared statements and indexes for common queries
- Data persistence: Database file stored in configurable location (default: ~/.gitdb/gitdb.duckdb)
- Transaction support: All write operations wrapped in transactions for consistency

### Performance considerations

- Index on repositories(full_name) for fast repository lookups
- Index on commits(repository_id, timestamp) for efficient commit history queries
- Index on files(repository_id, path) for quick file searches
- Automatic VACUUM scheduled after bulk operations
